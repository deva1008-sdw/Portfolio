1. Executive Summary & Goal

Modern application codebases contain deep dependency trees and nested imports. Standard vulnerability tools search flat locks but miss architectural flow vulnerabilities.

Cyber Nervous System (CNS) acts as a static analysis digital twin. It maps files, imports, classes, functions, and standard packages into a highly connected Graph Database, resolving structural calls and joining them directly against live CVE libraries via AI.

2. System Architecture

The platform utilizes a FastAPI backend running a tree-sitter ingestion worker, parsing raw codebase assets and writing nodes recursively to Neo4j Aura cloud instances.

4. Key Functional Implementations

AST Ingestion Engine

Parses standard Python code structures natively. Extracts namespaces, import linkages, class layouts, and function invocations, organizing them into logical parent-child relationships.

Dependency Security Matching

Integrates with OSV.dev APIs to query package dependencies. Automatically links matching security logs to the Graph node tree, identifying exposed call-paths.

AI-Driven Attack Path Mapping

Uses Gemini API endpoints to review call trees. Detects cross-boundary parameter injections and data leaks without requiring execution.

5. Cypher Schema Code Snippet

Below is the structured Neo4j database insertion query schema used to map dependencies dynamically:

// Cypher query to insert structural calls mapping code pathways
MERGE (p:Project {id: $projectId})
MERGE (f:File {path: $filePath})
MERGE (p)-[:HAS_FILE]->(f)
WITH f
UNWIND $dependencies AS dep
MERGE (d:Dependency {name: dep.name, version: dep.version})
MERGE (f)-[:IMPORTS]->(d)
WITH d
UNWIND dep.vulns AS vuln
MERGE (v:Vulnerability {cve: vuln.cve, severity: vuln.severity})
MERGE (d)-[:HAS_VULNERABILITY]->(v)
