1. Executive Summary & Goal

In security research, reconnaissance requires executing multiple slow tools consecutively. Manually sorting through thousands of ports, crawled URLs, and files is a major operational bottleneck.

BugBounty-AI automated this process by running core tools in asynchronous Celery workers and piping raw findings through LLMs to filter false positives and surface viable exploits.

2. System Architecture

A FastAPI endpoint receives target domains, schedules background processes in a Redis-brokered Celery queue, writes findings to PostgreSQL, and calls AI processors to evaluate targets.

3. Technology Stack Breakdown

Celery & Redis Job queuing broker for long-running scanner processes.
Offensive Tooling Subfinder (domains), HTTPX (ports), Katana (web crawling).
OpenAI / Ollama LLM APIs to analyze crawled responses for security anomalies.
FastAPI & Postgres Relational ledgers tracking target statuses and audit history.

4. Key Functional Implementations

Multi-Stage Scanner Chains

Enqueues sequential sub-scans: discovering hosts, testing active ports, scraping routes, and mapping code directories automatically.

LLM Payload Verification

Extracts HTML header metadata and error payloads. Passes them to an LLM evaluator to spot SQLi, XSS, or directory traversal leaks.

Dockerized Infrastructure

Designed a compose bundle to handle scanner binary installations, Python library requirements, and task brokers seamlessly.

Below is the Python Celery task blueprint that triggers active binary tool wrappers and logs status changes:

# Celery worker task executing subfinder binary wrapper
@shared_task(bind=True)
def run_subfinder_recon(self, target_id: str, domain: str):
    self.update_state(state='PROGRESS', meta={'stage': 'Subdomain Scanning'})
    cmd = ["subfinder", "-d", domain, "-silent", "-json"]
    result = subprocess.run(cmd, capture_output=True, text=True)
    
    subdomains = []
    for line in result.stdout.splitlines():
        data = json.loads(line)
        subdomains.append(data.get("host"))
        
    db = SessionLocal()
    save_recon_hosts(db, target_id, subdomains)
    db.close()
    return {"hosts_found": len(subdomains)}
