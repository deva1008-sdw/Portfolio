1. Executive Summary & Goal

Mobile background tracking systems suffer from aggressive OS battery managers, leading to dropped coordinates. Additionally, privacy-compliant tracking requires strict opt-in indicators.

SafeTracker is a consent-first GPS tracking framework combining an Android React Native client, an Express backend, and a Leaflet.js real-time browser monitor dashboard. It features robust offline buffering caches and stable hardware tracking identifiers.

2. Data Pipeline

GPS coordinates are recorded on the mobile client, queued locally during connection drops, batch uploaded, and broadcasted to browser map dashboards via WebSockets.

3. Technology Stack Breakdown

React Native & Expo Cross-platform GPS background fetching utilities.
Leaflet.js Maps Interactive map UI canvas displaying custom path coordinates.
Express & WebSockets Node.js server pushing locations directly to client browsers.
Local storage cache Queue logic buffering 2k offline track-points dynamically.

4. Key Functional Implementations

Consent-First Option Screens

Enforces active opt-in user confirmations and permanent status bar notification toggles, conforming to store privacy policies.

Battery-Efficient Offline Buffering

Stores raw locations locally during data gaps and uploads in optimized batch posts when networks restore to save device power.

Stable Device Fingers

Combines native hardware-linked IDs (`androidId` / `identifierForVendor`) to catalog device coordinates under stable tracks.

5. GPS Upload Code Snippet

Below is the Express location ingestion endpoint that accepts coordinate batches and broadcasts updates:

// Express endpoint for batch coordinates ingestion
app.post("/api/location/batch", async (req, res) => {
    const { deviceId, coordinates } = req.body;
    if (!deviceId || !Array.isArray(coordinates)) {
        return res.status(400).json({ error: "Missing parameters" });
    }
    
    // Log target and broadcast coordinates via ws
    broadcastToMonitors(deviceId, coordinates);
    
    // Save to server local file system ledger
    await saveLocationHistory(deviceId, coordinates);
    
    res.status(200).json({ status: "synced", count: coordinates.length });
});
